The Magic Of Recursion

Recursion is one of the most mind boggling concepts in computer science, which seems to magically solve problems, it works on a very simple process, but the semantics of it, and its execution quickly escalate to astonishing levels of complexity.

Mathematically, a recursive function is a function f(x) which is defined in terms of itself, that is the values generated by the function f(x) for a specific input x=n depend on a set of values generated by it for x<n(previous values of x). A classic example of this is the factorial function, if we wish to find the factorial of 5 ie 5!, we can simply multiply 5 with the value of 4!, but here, as we can see, we encounter another factoral function which demands us to know the value of 4!, proceeding as we did before, the list on the left represents the the recursive calls towards the base case, and the list on the right represents the returning of values by every recursive call, eventually stopping at the desired answer.

  • 5! = 5*4!
  • 4! = 4*3!
  • 3! = 3*2!
  • 2! = 2*1!
  • 1! = 1*0!
  • 0! = 1 (this is the only value that we know absolutely(base case))
  1. 0! = 1
  2. 1! = 1*0! = 1*1 = 1
  3. 2! = 2*1! = 2*1 = 2
  4. 3! = 3*2! = 3*2 = 6
  5. 4! = 4*3! = 4*6 = 24
  6. 5! = 5*4! = 5*24 = 120
